저의 장점

저의 전 직장에서의 경험과 프로젝트 시 경험을 돌아봤을 때
저의 장점은 정보를 얻고 그것을 적재적소에 사용하는 능력이 장점이라고 생각합니다

전 직장에서 프로젝트 중 회사 사정 때문에 주요 인원들이 회사를 나가서
혼자서만 장기간 외국 출장을 간 적이 있습니다
장비 작업 때 SW / HW 문제점이 발생하면 둘 다 혼자서 해결해야 할 상황 이었는데
모르는 문제가 발생 시 회사에 인원이 없어 물어볼 사람도 없었기에
최대한 직접 장비를 돌려서 얻은 정보와 인터넷과 남은 인원에게 정보를 수집해
그 정보를 활용해서 문제를 해결하고 장비를 개선 한 경험이 있습니다

또한 팀 프로젝트 하는 중에 다른 팀원이 저에게 문제점이나 구현이 막히는 부분을 물어보면
모르는 부분이라도 빠르게 해결법을 찾아서 해결 후 공유한 경험이 있습니다

그런 경험을 보았을 때 저는 무언가 막히거나 문제점이 생겼을때
제가 모르는 부분이라도 정보를 찾은 뒤 그 정보를 이용해 적재적소에 사용하는 능력이 있다고 생각합니다

또한 이렇게 해서 문제점을 해결했을 때 큰 성취감을 느낍니다.

프로젝트 완료 후 느낀점

1차 2차 팀 프로젝트 진행 시 느낀점으로는

깃(git)의 중요성입니다

지금까지 개발하면서 깃을 알고는 있었지만 제대로 활용을 하지 않았었고
심지어 이전 회사에서도 깃을 제대로 안 쓰는 기업이라 압축,다운로드 받으며 주먹구구식으로 진행했었는데
이번에 깃을 제대로 활용하면서 그 중요성을 확실히 느낄 수 있었습니다

버전 관리가 쉬워 현재 버전에 문제가 생길시 이전 버전을 쉽게 확인하고 돌아갈 수 있었으며
특히 팀원들 간의 협업과 코드 공유가 너무 편했습니다
물론 중간에 병합시 충돌이 나는 문제들 때문에 난감했지만
점차 사용해 가며 서로 충돌 날 것 같은 부분은 공유해 가며 숙달되니 괜찮았습니다

깃을 본격적으로 사용해보니 그전 깃을 활용 안 했던 게 후회될 정도였습니다
그래서 다음 프로젝트 부터 꼭 깃을 활용하고 사용하고 있습니다


개인프로젝트 진행시 느낀점으로는

WAS 서버와 API 서버 의 개발시 차이 입니다

두 팀 프로젝트에서는 Spring 프레임워크로 만든 톰캣 서버 하나로 콘텐츠 전달, db 작업등
WAS 서버가 하는 모든 역할을 다 했지만

개인프로젝트를 수행할때에는 윗 프로젝트들과 다르게
프론트엔드쪽은 웹서버(firebase)를 통해 따로 전달 및 호스팅을 하였고
백엔드쪽은 API 서버로서의 역할만 하도록 만들었습니다

프로젝트 진행하면서 겪은 일 중 하나는
해당 프로젝트에서는 2차 팀 프로젝트와 같이 Spring security를 이용해 보안을 구성하려 했는데
API 서버로써 활용해야 하기 때문에 2차 팀 프로젝트와 같이 세션을 이용 못 하는 상황이었습니다

그때 문득 생각난 게 2차 프로젝트에서 사용한 금융결제원 API 였고
살펴보니 토큰을 활용하는 방식을 사용하는 것을 확인
API서버로 개발시 인증은 jwt 를 이용하여 토큰 을 사용하는 방식으로 해야 된다는 것을 깨달았습니다

그러면서 세부적인 세팅이나 코드도 많이 달라지는 것을 보며
WAS 서버로 개발할 때와 API 서버로 개발할 때와의 차이를 알게 되었습니다


팀 프로젝트와 개인 프로젝트를 진행한 후 느낀점은

팀원의 유무에 따른 마인드의 차이점 입니다

프로젝트를 같이 하는 팀원이 있을 때는 프로젝트 진행 시 막혔을 때나 프로젝트 방향성의 의문점이 생겼을 때
따로 회의시간을 잡아 이야기를 나누거나 시간 있는 팀원과 대화를 나누거나 하는데
개인 프로젝트시 에는 그렇게 할 수 없어
위의 상황이 되었을 때 혼자 고민하는 시간이 필요해
효율이 떨어지는 것을 느꼈습니다
